# 28-Day Algorithm Plan

Practice one main topic each day. The point is not to finish many problems, but to understand the pattern well enough to recognize it again.

## Week 1: Foundations

### Day 1: Arrays and Simulation

- Focus: indexing, traversal, boundary handling
- Problem types: remove element, merge arrays, spiral traversal

### Day 2: Hash Map and Hash Set

- Focus: counting, deduplication, complement lookup
- Problem types: two sum, valid anagram, longest consecutive sequence

### Day 3: Two Pointers

- Focus: opposite-direction and same-direction pointer movement
- Problem types: move zeroes, valid palindrome, container with most water

### Day 4: Sliding Window

- Focus: window expansion, shrink conditions, frequency map
- Problem types: longest substring without repeat, minimum size subarray sum

### Day 5: Prefix Sum

- Focus: range sum conversion, count tricks with maps
- Problem types: subarray sum equals k, pivot index

### Day 6: Binary Search

- Focus: monotonicity, left boundary, right boundary
- Problem types: search insert position, first bad version, search rotated array

### Day 7: Review Day

- Focus: redo 2 to 3 missed problems from days 1 to 6
- Output: write a one-page summary of common bugs

## Week 2: Linked Structures and Trees

### Day 8: Linked List Basics

- Focus: dummy node, pointer rewiring, fast and slow pointers
- Problem types: reverse list, merge two sorted lists, remove nth from end

### Day 9: Stack

- Focus: last in first out, monotonic stack basics
- Problem types: valid parentheses, daily temperatures, min stack

### Day 10: Queue and Deque

- Focus: bfs support, monotonic deque ideas
- Problem types: sliding window maximum, moving average

### Day 11: Binary Tree Traversal

- Focus: preorder, inorder, postorder, level order
- Problem types: max depth, invert tree, same tree

### Day 12: Binary Search Tree

- Focus: ordered property, validate/search/insert logic
- Problem types: validate BST, kth smallest in BST

### Day 13: Tree Recursion and Divide-and-Conquer

- Focus: what each recursive call returns
- Problem types: diameter of binary tree, balanced binary tree, lowest common ancestor

### Day 14: Review Day

- Focus: redo one linked list problem and two tree problems
- Output: summarize recursion templates

## Week 3: Backtracking, Heap, and Graph Start

### Day 15: Backtracking Basics

- Focus: path, choice list, undo step
- Problem types: subsets, permutations, combinations

### Day 16: Advanced Backtracking

- Focus: pruning, duplicate handling
- Problem types: combination sum, palindrome partitioning, word search

### Day 17: Heap and Priority Queue

- Focus: top-k, streaming minimum or maximum
- Problem types: kth largest element, top k frequent elements, merge k sorted lists

### Day 18: Intervals

- Focus: sorting by start or end, merge logic
- Problem types: merge intervals, insert interval, meeting rooms

### Day 19: Graph Traversal

- Focus: graph representation, dfs, bfs, visited set
- Problem types: number of islands, clone graph, flood fill

### Day 20: Topological Sort

- Focus: indegree, dependency graph
- Problem types: course schedule, course schedule II

### Day 21: Review Day

- Focus: redo one backtracking problem and two graph problems
- Output: write down dfs vs bfs decision rules

## Week 4: Dynamic Programming and Interview Finishers

### Day 22: DP Basics

- Focus: state, transition, base case
- Problem types: climbing stairs, house robber, min cost climbing stairs

### Day 23: 1D DP

- Focus: take or skip decisions
- Problem types: coin change, maximum subarray, word break

### Day 24: 2D DP

- Focus: grid transition and table meaning
- Problem types: unique paths, minimum path sum, longest common subsequence

### Day 25: Greedy

- Focus: local choice and proof intuition
- Problem types: jump game, gas station, partition labels

### Day 26: Bit Manipulation

- Focus: xor, bit mask, state compression basics
- Problem types: single number, counting bits, reverse bits

### Day 27: Mixed Mock Interview Day

- Focus: solve 2 medium problems under time pressure
- Suggested mix: one data structure problem and one algorithm problem

### Day 28: Final Review Day

- Focus: review all weak areas from the month
- Output: make your top 10 mistake list and top 10 reusable patterns

## Difficulty suggestion

- Days 1 to 10: Easy + Medium
- Days 11 to 21: Mostly Medium
- Days 22 to 28: Medium, with a few Hard previews only if time allows

## Repetition rule

For every problem you could not solve smoothly:

- Review after 2 days
- Review after 7 days
- Review after 14 days

## What to record each day

- What pattern did this problem test
- What clue told you which pattern to use
- What bug or confusion slowed you down
- Can you explain the time and space complexity clearly
